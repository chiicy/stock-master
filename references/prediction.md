# 预判与操作建议框架

## 使用原则

- 预判是对当前信息的条件性推演，不是承诺，也不是“明天一定怎样”的断言。
- 先汇总一致性信号，再列出矛盾信号，最后给时间切片判断。
- 用户问“未来三天怎么看”“短线怎么操作”“像虾评一样预测”时，按本框架输出。
- 只做方向性判断：上涨倾向 / 震荡倾向 / 回落倾向；不要做伪精确点位预测。

## 分析维度

### 1. 综合判断
- 多视角一致性
- 哪些信号支持看多
- 哪些信号提示风险
- 明确哪些结论来自技术面，哪些来自资金 / 板块 / 基本面补强或拖累

### 2. 时间切片
- T+1
- T+2 ~ T+3
- 中线观察位
- T+1 更看短线强弱和关键位争夺
- T+2 ~ T+3 更看方向是否延续或回到震荡
- 中线观察位用于判断短线扰动是否升级成趋势改变

### 3. 关键价位
- 第一支撑 / 压力
- 第二支撑 / 压力
- 失效位
- 若缺口信息不完整，可把“未回补缺口附近”作为辅助观察位，但不要编造精确价格

### 4. 操作建议
- 激进
- 稳健
- 保守
- 激进：更重视突破和弹性，但要加仓位 / 追价条件
- 稳健：更重视回踩确认和量价配合
- 保守：更重视基本面完整度、趋势稳定度和回撤控制

## 输出模板

### 预判结论
- 基准判断：震荡偏多 / 震荡偏弱 / 延续趋势 / 等待企稳
- 主要依据：2~4 条，必须写出支撑该判断的证据
- T+1：偏强 / 反复 / 承压
- T+2 ~ T+3：上涨倾向 / 震荡倾向 / 回落倾向
- 关键观察点：1~3 条
- 失效条件：跌破 / 突破哪些位置，或出现哪些指标反转
- 操作建议：按风险偏好区分，避免绝对化措辞

## 写作要求
- 预测必须注明依据
- 不要用确定性语气
- 预判是对当前信息的条件性推演，不是承诺
- 若资金面、板块面、消息面与技术面矛盾，要把矛盾点单列出来
- 失效条件不能只写“走势变差”，要写清可观察的价位或信号
